## BUG REPORT

### BUG-1 [HIGH] — B6: approval_snapshots not truly immutable
**Location:** `supabase/migrations/023_publisher_shield.sql`
**Problem:** RLS policy `CREATE POLICY "tenant_isolation" ON approval_snapshots USING (...)` applies to ALL operations (SELECT, UPDATE, DELETE). A tenant can mutate or delete their own approval snapshots — defeating the legal audit trail.
**Expected:** Insert-only. Only SELECT and INSERT should be allowed.
**Fix:**
```sql
-- Drop the ALL policy
DROP POLICY "tenant_isolation" ON approval_snapshots;

-- Replace with explicit SELECT + INSERT only
CREATE POLICY "select_own" ON approval_snapshots
  FOR SELECT USING (tenant_id = current_setting('app.tenant_id', true)::uuid);

CREATE POLICY "insert_own" ON approval_snapshots
  FOR INSERT WITH CHECK (tenant_id = current_setting('app.tenant_id', true)::uuid);

-- No UPDATE or DELETE policy = those operations are blocked by default
```
**Migration needed:** Yes — migration 040.

### BUG-2 [HIGH] — F2: Free plan breaker threshold dead code
**Location:** `apps/api/src/billing/pricing.ts` lines 16 + 80
**Problem:** `PLAN_PRICING.free.breakerFreezePct = 30` is defined but never read. `breakerState()` uses `BREAKER.freezeAtPct = 40` for ALL plans. Free plan users should freeze at 30% of fee, not 40%. This means free users can consume 33% more AI than intended.
**Impact:** Vigmis loses money on every free-plan customer with high AI usage.
**Fix:**
```typescript
// In breakerState(), use plan-specific threshold:
export function breakerState(
  plan: Plan,
  feeUsd: number,
  aiCostUsd: number,
): 'ok' | 'degrade' | 'freeze' {
  if (feeUsd <= 0) return aiCostUsd > 0 ? 'freeze' : 'ok';
  const pct = (aiCostUsd / feeUsd) * 100;
  const freezeAt = PLAN_PRICING[plan].breakerFreezePct;  // use plan-specific
  if (pct >= freezeAt) return 'freeze';
  if (pct >= BREAKER.degradeAtPct) return 'degrade';
  return 'ok';
}
```
**Note:** The alert email in `usage.ts` already correctly says 30% for free — only the enforcement was wrong.

### BUG-3 [BLOCKER] — J3: CRON_SECRET hardcoded default
**Location:** All 24 cron routes — e.g. `apps/web/app/api/cron/briefings/route.ts` line 6
**Problem:** `const cronSecret = process.env.CRON_SECRET ?? 'vigmis-cron';` — anyone who reads the source code knows the default secret. If `CRON_SECRET` is not set in production, all crons are publicly callable with `Authorization: Bearer vigmis-cron`.
**Fix:**
```typescript
// Throw in production if CRON_SECRET is not set:
const cronSecret = process.env.CRON_SECRET;
if (!cronSecret) {
  return new Response('Server misconfiguration', { status: 500 });
}
```
**Note:** Verify CRON_SECRET is set in all environments (Vercel env vars) before deploying this fix.

### BUG-4 [MEDIUM] — A6: Missing tsconfig.json in packages
**Location:** `packages/ad-connectors/` and `packages/ai-router/`
**Problem:** Neither package has a `tsconfig.json`. The `lint` script runs `tsc --noEmit` which fails with no tsconfig (shows TypeScript help text instead of checking types). TypeScript errors in these packages are silently missed.
**Impact:** Type errors in ad connector or AI router code are never caught by CI.
**Fix:** Add tsconfig.json to each package:
```json
{
  "compilerOptions": {
    "target": "ES2022",
    "module": "ESNext",
    "moduleResolution": "bundler",
    "strict": true,
    "noEmit": true,
    "skipLibCheck": true
  },
  "include": ["src"]
}
```
